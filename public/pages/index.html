La page vierge/modele

Quand on accède à une page vierge, ce qui s'affiche dans le navigateur est un fichier HTML "modele" par defaut.

On peut acceder au code de ce fichier "par defaut" en cliquant sur "::source" sur une page vierge.

Ce fichier par defaut, visuellement vide, contient le squelette de base d'une page HTML standard.

Une page HTML standard comporte une section <head> et une section <body>.

<head>

La section <head> sert à donner des "meta" instructions au navigateur qui accède à une page, par exemple :

  • lien vers un fichier CSS
    <link rel="stylesheet" type="text/css" href="/css/style.css" >
    c'est de ce fichier que vient le style par defaut des pages cahut, vous pouvez le modifier en allant à l'URL /css/style.css::source ou bien en creer un autre par exemple en allant sur l'URL /css/style2.css::source. Il faudra alors inclure ce nouveau fichier en ajoutant la ligne
    <link rel="stylesheet" type="text/css" href="/css/style2.css" >
    dans le <head>. Vous pouvez inclure autant de fichier CSS que vous voulez dans une page (un seul suffit normalement, mais bref voilà c'est possible).

  • lien vers un fichier favicon.ico (pour mettre une icone dans l'onglet)
    <link rel="icon" type="image/x-icon" href="/favicon.ico" />
    vous pouvez mettre une image que vous voulez utiliser comme favicon à l'adresse "/favicon.ico"

  • lien vers un fichier script (javascript)
    <script src="/js/script.js">
    ce fichier est vide pour le moment, mais si un jour vous apprenez à vous servir du javascript... ben voilà c'est là qu'il faudra le mettre

  • une balise "title" qui correspond à ce qui est écrit dans l'onglet
    <title>Arthur</title>

Le <head> de cette page vierge contient aussi deux lignes :

<link rel="stylesheet" type="text/css" href="/interface/css/simpleEdit.css" />

<script type="module" src="/interface/js/simpleEdit.js" async=""></script>

La première est un lien vers un fichier CSS qui permet de styliser la zone de texte/edition à gauche quand on est connecté.

La seconde est un lien vers un fichier Javascript, il entre en jeu quand on écrit dans la textarea, c'est lui qui gère l'affichage dans la zone #content, qui gère la sauvegarde en temps réel de la page, et pas mal d'autre petits details.

Ces deux fichiers sont dans le dossier /interface/, tout ce qu'il y a dans ce dossier /interface/ c'est que des trucs "necessaires" au fonctionnement de cahut, ils sont disponibles publiquement mais il vaut mieux éviter d'y toucher si on sait pas trop ce qu'on fait.

<body>

en règle générale, le <body> d'une page html contient le "contenu" de cette page, destiné aux "humains"

dans le <body> d'une page vierge sur cahut, il y a 2 "entitées" :

  1. une div "interface"
  2. une div "content"

la div "interface" contient les éléments d'"interface", dont la zone de texte à gauche, avec les boutons "hide", "::source", etc.. (d'autre boutons à venir)

La div "content" correspond à la zone centrale dans laquelle s'affiche ce qu'on tape dans la zone de texte à gauche, sur une page vierge cette div est vide

Créer ses propres pages modèles

Il est possible sur cahut de creer ses propres pages modeles.

pour ça il faut aller à une URL type :

/modele
toutes les pages vierges qui commencent par / utiliseront /modele comme modèle

ou bien

/poesie/modele
toutes les pages vierges qui commencent par /poesie/ utiliseront /poesie/modele comme modèle

ou bien

/mammifère/lapins/modele
toutes les pages vierges qui commencent par /mammifère/lapins/ utiliseront /mammifère/lapins/modele comme modèle

Vous pouvez modifier completement ces pages modèles, y compris le <head> via modele::source, ça veut dire que vous pouvez creer des pages modeles avec un fichier CSS dedié par exemple /css/mammifere/lapins/style.css, sans oublier de rajouter <link rel="stylesheet" type="text/css" href="/css/mammifere/lapins/style.css" > au <head>

Des pages privées

J'ai créé un petit système pour permettre de faire des pages privées.

Ces pages privées sont vierges si on est pas connecté, il faut être connecté pour pouvoir acceder à son contenu.

Pour qu'une page soit privée, il suffit que son adresse finisse par "::hidden"

Exemple :

/::hidden

/lapin::hidden

/mammifère/chevre::hidden

Ca peut être pratique en particulier pour faire des sommaires cachés vers des pages que vous ne voulez pas specialement partager.

On defini la police qu'utilise un texte sur internet via le CSS. De la même façon qu'on peut definir la couleur d'un texte en utilisant :

color:"couleur";

on peut definir la police qu'utilise un texte via :

font-family:"nom-d'une-Police-1", "nom-d'une-Police-2";

(si la Police-1 n'est pas dispo, le navigateur va essayer d'utiliser la Police-2, etc...)

Polices génériques :

Il existe des familles de police "par defaut" qui sont disponibles sur tout les ordinateurs, on les appeles les "familles génériques", il y a par exemple :

1. font-family: serif;

Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.

2. font-family: sans-serif;

Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.

3. font-family: monospace;

Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.

Dans la famille monospace tous les caractères font la même largeur (un i prend autant d'espace qu'un W)

Les serifs correspondent aux petites fioritures aux extremitées des caratères (voir page wikipedia serif)

Ces polices génériques utilisent les polices déjà présentent par defaut dans l'ordinateur de celui qui visite la page web. Comme il existe des centaines de polices "serif" ou "sans-serif", on ne peut pas être sûr du resultat selon l'ordinateur qui visite la page la page ne s'affichera pas de la même façon si on utilise un Mac ou Windows ou Linux.

Polices Personalisées

Heureusement on peut utiliser des polices personnalisées, il faut pour ça que le site sache où trouver les fichiers de la police.

Il y a plusieurs solutions pour ça :

1. google font

c'est assez pratique, la facilité, google se charge de mettre à disposition les fichiers de police. On peut selectionner plusieurs polices differentes. Ensuite google nous donne un bout de code à incorporer au "head" de la page ou bien dans le CSS, enfin on peut utiliser ces polices via la consigne CSS "font-family".

2. Heberger soi-même les fichiers de police

Comme c'est le cas pour la police par defaut de cahut (Linux Libertine), les fichiers permettant d'utiliser la police sont stoqués sur le serveur de votre cahut dans le dossier "/CSS/fonts/" (accessible via "/public/css/fonts/").

On peut techniquement stoquer ces fichiers police n'importe où sur le serveur, mais c'est plus sage de les stoquer dans ce dossier "fonts" (surtout qu'il y a possibilité de faire des sous-dossiers par police).

Ces fichiers peuvent être de differents formats, les formats les plus conseillés sont TTF/OTF (meilleure qualitée mais lentes à charger) et WOFF (format compressé)

Il y a souvent plusieurs fichiers pour la même police, qui correspondent aux variantes gras (avec desfois plusieurs poids de gras) ou italique ou autre..

Une fois qu'on a mis en ligne les fichiers de police, il faut les "déclarer" au CSS : il faut que le CSS sache où sont les fichiers et qu'il sache sous quel(s) nom(s) on va utiliser la police.
Pour ça on utilise la directive @font-face, exemple :


@font-face {
    font-family: 'le nom de la police pour s'en servir';
    src: url('/css/fonts/fichierDePolice.ttf') format('ttf');
    font-weight: normal; 
    font-style: normal;

}

Il faut utiliser cette directive au debut du CSS (idealement au debut du fichier style.css)

le site fontsquirrel propose un "generateur de webfont" qui génère automatiquement le code CSS pour "declarer" les polices en CSS. Il génère aussi les fichiers adaptés (en mode expert on peut choisir sous quel format on veut les fichiers police (TTF/WOFF))

Quand on commence à écrire sur cette page vierge (par exemple /mammifere/lapins/nain), le code de cette page HTML "par defaut" est mis à jour selon ce qu'on entre dans la zone de texte puis envoyé au serveur qui va la sauvegarder dans le fichier /pages/lapin.html.

C'est le script "/interface/js/simpleEdit.js" qui gère la mise à jour de la page et qui l'envoit au serveur pour sauvegarde

Le bouton "source" permet de lire et modifier le code source de cette page lapin.html
